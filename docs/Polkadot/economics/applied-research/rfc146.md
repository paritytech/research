# Deflationary Transaction Fee Model for the Relay Chain and its System Parachains (accepted): RFC-0146 

|                 |                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------- |
| **Start Date**  | 20th May 2025                                                            |
| **Description** | This RFC proposes burning 80% of transaction fees on the Relay Chain and its system parachains, adding to the network's existing deflationary capacity.     |


## Summary

This RFC proposes **burning 80% of transaction fees** accrued on Polkadot’s **Relay Chain** and, more significantly, all its **system parachains**. The remaining 20% would continue to incentivize validators on the Relay Chain and collators on system parachains for including transactions. The motivation behind the 80:20 split is to preserve validator incentives, which are crucial for the security of the network. It should also help establish a consistent fee policy across the Relay Chain and all system parachains.

* On the **Relay Chain**, the change redirects the share that currently goes to the Treasury toward burning. Considering the move toward a [minimal Relay](https://polkadot-fellows.github.io/RFCs/approved/0032-minimal-relay.html) ratified by RFC0032, a change to the fee policy will likely be symbolic for the future, yet it contributes to overall coherence.

* On **system parachains**, the collator share would be reduced from 100% to 20%, with 80% burned. Since collator rewards do not significantly contribute to the shared security model, this adjustment should not affect the network's integrity.

The proposal reinforces the system's **deflationary direction**, enabling direct value capture for DOT holders through increased network activity.

## Motivation

To date, transaction fees on the Relay Chain and system parachains have, with a few exceptions, been relatively low. This choice is by design, as Polkadot is built to scale and offer low-cost transactions. While this principle has not changed, growing network activity could still lead to a meaningful accumulation of fees over time. 

Implementing this RFC ensures that increasing activity and the resulting fees are captured for all token holders. It further aligns with the way the network handles transaction, or coretime-usage, fees. The rationale behind this decision is similar to that outlined in [RFC0010](https://polkadot-fellows.github.io/RFCs/approved/0010-burn-coretime-revenue.html). Burning transaction fees has the following benefits:

* **Compensation for Coretime Usage.** System parachains do not participate in open-market bidding for coretime. Instead, they hold a special governance status that allows them to consume network resources without explicitly paying for them. Burning transaction fees serves as a simple and effective way to compensate for the revenue that would otherwise be generated in the open market.

* **Value Accrual and Deflationary Pressure.** By burning transaction fees, the system effectively reduces the token supply, increasing the scarcity of the native token. This deflationary pressure can enhance the token's long-term value and ensure that captured value is distributed equally among all existing token holders. The proposal requires minimal code changes, making it inexpensive to implement. Yet, it introduces a consistent policy for handling transaction fees across the network. The proposal also prepares Polkadot for a future in which fee burning could serve as a counterweight to an inflationary token model, ensuring that the value generated by network usage is returned to all DOT holders.

## Stakeholders

* **All DOT Token Holders** benefit from reduced supply and direct value capture as network usage increases.

* **System Parachain Collators.** This proposal effectively reduces the income earned by system parachain collators. The impact on the status quo is negligible, as fees earned by collators have been minimal, around $1,300 monthly across all system parachains (based on data collected between November 2024 and April 2025). Most of their compensation comes from Treasury reimbursements distributed through bounties. As such, this change should not have any meaningful effect on collator incentives or behavior.

* **Validators** remain unaffected, as their rewards stay unchanged.


## Sidenote: Fee Assets

Some system parachains may accept other assets deemed **sufficient** to cover transaction fees. This has no impact on the proposal, as the **asset conversion pallet** ensures that DOT is used to pay the fees, which can then be burned.
